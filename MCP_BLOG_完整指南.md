## 🚀 快速开始

### 1. 启动你的博客系统
```bash
# 启动后端API
cd server
npm run dev

# 启动前端（新终端）
npm run dev
```

### 2. 获取JWT令牌
访问 http://localhost:5173 登录后，在浏览器开发者工具的localStorage中找到token。

### 3. 在Kiro中使用

#### 设置认证令牌
```
请帮我设置博客认证令牌：eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

#### 创建并发布文章
```
请帮我创建一篇文章：
标题：Vue 3 Composition API 最佳实践
内容：在现代前端开发中，Vue 3的Composition API为我们提供了更灵活的组件逻辑组织方式。本文将深入探讨其最佳实践...
立即发布：true
```

#### 管理现有文章
```
请显示我的所有草稿文章，然后发布ID为2的文章
```

## 🔧 可用的MCP工具

| 工具名称 | 描述 | 参数 |
|---------|------|------|
| `create_post` | 创建新文章 | title(必需), content, published |
| `update_post` | 更新现有文章 | id(必需), title, content, published |
| `publish_post` | 发布文章 | id(必需) |
| `list_posts` | 获取文章列表 | published(可选) |
| `set_auth_token` | 设置认证令牌 | token(必需) |

## 🛠️ 故障排除

### 常见问题

**Q: MCP服务器无法连接**
A: 检查 `.kiro/settings/mcp.json` 配置和文件路径是否正确

**Q: 认证失败**
A: 确保JWT令牌有效且未过期，可以重新登录获取新令牌

**Q: API连接失败**
A: 确保博客服务器在 http://localhost:3000 正常运行

**Q: 权限不足**
A: 检查用户是否有权限操作该文章（只能操作自己的文章）

### 调试步骤
1. 检查MCP服务器构建：`cd mcp-blog-server && npm run build`
2. 测试API连接：`curl http://localhost:3000/api/posts`
3. 在Kiro中查看MCP面板的连接状态
