# Markdown开发工具使用指南

## 概述

Markdown开发工具是一个专为开发环境设计的性能监控和测试工具，帮助开发者监控和优化Markdown渲染性能。

## 功能特性

### 🎛️ 手动控制显隐
- **切换按钮**：右上角的圆形按钮可以显示/隐藏工具面板
- **最小化功能**：面板可以最小化，只显示标题栏
- **状态记忆**：工具会记住您的显示偏好设置

### ⌨️ 快捷键支持
- **Ctrl/Cmd + Shift + D**：快速切换工具面板显隐
- 支持Windows、Mac和Linux系统

### 📊 实时性能监控
- **渲染统计**：总渲染次数、缓存命中/未命中
- **性能指标**：平均/最大/最小渲染时间
- **缓存状态**：缓存大小、命中率、过期时间
- **自动更新**：每2秒自动刷新统计数据

### 🧪 测试工具
- **性能测试**：测试Markdown渲染性能和缓存效果
- **缓存测试**：验证缓存功能是否正常工作
- **实时渲染测试**：输入Markdown内容实时查看渲染效果
- **报告导出**：导出详细的性能分析报告

## 使用方法

### 1. 显示/隐藏工具

**方法一：点击按钮**
- 点击右上角的🔧按钮显示工具面板
- 再次点击或点击面板右上角的✕按钮隐藏面板

**方法二：使用快捷键**
- 按下 `Ctrl + Shift + D` (Windows/Linux)
- 按下 `Cmd + Shift + D` (Mac)

### 2. 最小化面板

- 点击面板标题栏右侧的📕按钮最小化面板
- 点击📖按钮展开面板

### 3. 查看性能统计

工具面板包含以下统计信息：

**性能统计**
- 总渲染次数：Markdown内容被渲染的总次数
- 缓存命中：从缓存中获取结果的次数
- 缓存未命中：需要重新渲染的次数
- 缓存命中率：缓存效率百分比
- 平均渲染时间：每次渲染的平均耗时
- 最大渲染时间：单次渲染的最长耗时

**缓存信息**
- 缓存大小：当前缓存的条目数量
- 过期时间：缓存条目的过期时间（分钟）

### 4. 运行测试

**性能测试**
1. 点击"性能测试"按钮
2. 工具会自动运行多次渲染测试
3. 查看测试结果，了解缓存带来的性能提升

**缓存测试**
1. 点击"缓存测试"按钮
2. 验证缓存功能是否正常工作
3. 查看缓存统计信息

**实时渲染测试**
1. 在文本框中输入Markdown内容
2. 实时查看渲染效果
3. 测试不同的Markdown语法

### 5. 管理缓存和指标

**清除缓存**
- 点击"清除缓存"按钮清空所有缓存数据
- 用于测试或释放内存

**清除指标**
- 点击"清除指标"按钮重置所有性能统计
- 用于重新开始性能监控

**导出报告**
- 点击"导出报告"按钮下载JSON格式的性能报告
- 包含详细的统计数据和最近的性能指标

## 界面说明

### 切换按钮状态
- **🔧 灰色**：工具面板已隐藏
- **🔧 绿色**：工具面板已显示
- **悬停效果**：按钮会轻微放大

### 面板控制
- **📕 最小化**：隐藏面板内容，只保留标题栏
- **📖 展开**：显示完整面板内容
- **✕ 关闭**：完全隐藏工具面板

### 帮助提示
- 鼠标悬停在按钮上会显示操作提示
- 测试工具标题旁有快捷键提示

## 性能优化建议

### 根据统计数据优化

1. **缓存命中率低于80%**
   - 检查内容是否频繁变化
   - 考虑增加缓存大小或延长过期时间

2. **平均渲染时间过长**
   - 检查Markdown内容复杂度
   - 考虑分段渲染大文档

3. **缓存大小接近上限**
   - 定期清理缓存
   - 优化内容重复度

### 最佳实践

1. **开发阶段**：保持工具开启，监控性能变化
2. **测试阶段**：运行完整测试，验证优化效果
3. **生产部署前**：导出性能报告，确保满足要求

## 注意事项

### 环境限制
- 仅在开发环境中显示（`import.meta.env.DEV`）
- 生产环境自动隐藏，不影响用户体验

### 浏览器兼容性
- 支持现代浏览器（Chrome 60+、Firefox 55+、Safari 12+）
- 使用localStorage保存状态，需要浏览器支持

### 性能影响
- 工具本身对性能影响极小
- 统计数据收集不会影响正常渲染
- 可以随时关闭以节省资源

## 故障排除

### 工具不显示
1. 确认当前为开发环境
2. 检查浏览器控制台是否有错误
3. 尝试刷新页面

### 快捷键不工作
1. 确认焦点在页面上（不在输入框中）
2. 检查是否与其他快捷键冲突
3. 尝试点击按钮方式

### 统计数据异常
1. 点击"清除指标"重置数据
2. 刷新页面重新开始监控
3. 检查是否有其他代码影响
